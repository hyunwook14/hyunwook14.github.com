JAVA

JAVA 란

자바는 썬 마이크로시스템즈 개발한 객체지향언어다. 자바는 운영체제에 독립적이라 자바로 만들어진 프로그램은 어떤 운영체제든 변경하지 않고 이용이 가능한 장점이 있다. 인터넷과 같은 환경에 적합하다.

역사

썬 엔지니어들이 Oak 언어 와 C++ 언어의 장점 및 단점을 보완하는 JAVA 를 개발했다. JAVA 는 처음에 PDA 같은 소형 가전제품에 사용할 소프트웨어용으로 개발하려고 하였으나 여러 운영체제를 사용하는 컴퓨터들이 통신하는 인터넷이 나오면서 운영체제에서 독립적으로 사용하는 특징이 있어 개발방향을 바꿨다. 처음에 인터넷에 사용된 자바로 만들어진 Applet은 정적인 웹페이지로 사운드, 애니메이션 등의 멀티미디어적 요소로 인기를 끌었지만 보완상의 문제로 Servelt 과 JSP과 같은 서버쪽 프로그래밍을 더 많이 사용한다.

특징

  • 운영체제에 독릭접 - JVM(자바가상머신)이 프로그램을 동작할 수 있게 운영체제가 알아들을 수 있는 언어로 변경하여 소통한다.
  • 객체지향언어 - 캡슐화, 다형성, 상속 등으로 재사용성과 유지보수에 용이하다.
  • 네트워크와 분산처리 - 네트워크 프로그래밍 라이브러리 (API) 제공해 쉽게 개발가능하다.
  • 멀티쓰레드 - 운영체제에 맞춰 멀티쓰레드를 구현하는게 다른데 JAVA로 만들어진 프로그램은 운영체제에 관계없이 구현가능하다.
  • 메모리관리 - 가비지켈럭터가 메모리를 관리해줘 프로그래머가 메모리를 신경 안쓰고 프로그래밍에 집중할 수 있다.
  • 동적로딩 - 자바로 작성된 어플리케이션은 클래스 여러개로 구성되어 있고 실행시 메모리에 클래스를 모두 올리는게 아니라 필요한 시점에 클래스를 메모리에 동적으로 올려 사용한다.

JVM

Java Virtual Machine으로 한국어로 직역하면 자바가상머신이다. 가상머신은 소프웨어로 만들어진 하드웨어로 소프트웨어로 구현된 컴퓨터로 생각하면 된다. Java가 운영체제에 독립적인 이유는 JVM에서만 동작하기때문이다. JVM은 동작할 기능들을 운영체제가 이해할 수 있는 언어로 번역하는 작업을 해 속도가 느린 단점이 있는데 최근에 이 단점도 바이트코드를 기계어로 번역하는 JIT컴파일로 덕분에 속도가 개선되었다. JVM은 운영체제에 종속되어있어 썬 마이크로시스템즈에서 운영체제에 맞는 소프트웨어를 제공한다.
